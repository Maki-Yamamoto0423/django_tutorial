高度なチュートリアル: 再利用可能アプリの書き方

Djangoでの名前空間の具体例

① テンプレートの名前空間

×　NG例

```
polls/templates/index.html
```

◯　良い例

```
polls/templates/polls/index.html
```

上記のようにすることで、コードでは、pollsという名前空間に属するindex.htmlだと明確になる

```
return render(request, "polls/index.html")
```

② 静的ファイルの名前空間

×　NG例

```
polls/static/style.css
```

◯　良い例

```
polls/static/polls/style.css
```

<link rel="stylesheet" href="{% static 'polls/style.css' %}">
→ 他アプリの style.css と衝突しない

③ URL の名前空間
📂polls/urls.py

```
app_name = "polls"

urlpatterns = [
    path("<int:pk>/", views.DetailView.as_view(), name="detail"),
]
```

使う側：

```
<a href="{% url 'polls:detail' pk=1 %}">詳細</a>
```

→ detail という名前が、polls アプリのものだと分かる

RESTful API について

それぞれの違い

REST API：RESTを意識して作ったAPI
RESTful API：RESTの思想をかなり忠実に守ってるAPI

RESTfulっぽい例  
・URLは「名詞」  
・操作はHTTPメソッドで表現  

```
GET /users → ユーザー一覧取得
GET /users/1 → ユーザー1件取得
POST /users → ユーザー作成
PUT /users/1 → ユーザー更新
DELETE /users/1 → ユーザー削除
```

RESTっぽいけどRESTfulじゃない例  
・動詞がURLに入ってる  
・HTTPメソッドの意味を活かしてない  

```
POST /getUser
POST /createUser
```

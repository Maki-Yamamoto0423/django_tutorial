Django Test Client とは？
自分が作成したWebアプリに対して「リクエストを送り、レスポンスを受け取る」
という一連の流れを、ブラウザを使わずにテストするためのツール

なぜ「擬似」なのか？
通常のブラウザ（Chromeなど）は、ネットワーク（HTTP）を介してサーバーにアクセスしますが、Test Client は Django 内部のコードを直接呼び出します。

本物のブラウザ： ブラウザ -> ネットワーク -> Webサーバー -> Django
Django Test Client： Pythonコード -> Django

何ができるのか？
GET や POST リクエストの送信: フォーム送信やページ遷移のシミュレーション
ログイン状態のシミュレート: client.login() で簡単にログイン済みユーザーとして振る舞える
レスポンスの検証: 戻ってきた HTML の内容、ステータスコード（200 OK や 404 Not Found）、使用されたテンプレートなどをチェックできる
リダイレクトの追跡: ページ移動が正しく行われたかを確認できる

Django の include() は、「URL 設定の丸投げ（委託）」 を行う関数

なぜ include が必要なのか？
もし include がなかったら、すべてのURL設定（投票アプリ、ブログ、ショッピング機能など）を１つの urls.py に書かなければならない

問題点：コードが膨大になり、管理が大変になる
解決策：include を使って、アプリごとに URL 設定を切り分ける

使い分け
自分で作ったアプリの URL を繋ぐときは必ず include() を使う
そのため、admin.site.urlsはinclude()する必要なし
